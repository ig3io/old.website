PyCharm And PEP8

First, I've got to say that I'm very happy with PyCharm and it's capacity to ease my day-to-day development with Python (I'm still amazed of how working with a Windows box became comfortable).

That being said, PEP8 support could be better. I'm not talking about style-error highlighting. I'm talking about the formatter un-PEP-izing my code.

Then I found that I could easily configure the multi-line method calls wrapping (my main problem) through File > Settings > Style > Python.

So I'm kind of a jerk for complaining.

As per PEP8 checking and warnings, I don't know if there's a more elegant way, but for now it make wonders. Thanks to the people at Ocean Observatories Iniciative and their Confluence wiki.

Replicating the instructions, we have:

  1. Go to File > Settings > External Tools
  2. Click to add a new one
  3. Name, group and description of your choice (only name is required)
  4. I left the checkbox options unchanged
  5. Program: route to pep8 executable (e.g. C:\Python27\Scripts\pep8.exe in my setup)
  6. Parameters: $FilePath$ (can be selected through Insert Macro)
  7. Working Directory: $ProjectFileDir$ (can be selected via Insert Macro)

With this, you can now go to Tools and execute pep8 and see in the IDE console the warnings generated (if any). Is not specially pretty, but it's not ugly.
